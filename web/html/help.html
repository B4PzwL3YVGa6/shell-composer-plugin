Composer Help

Composer is an IBM Cloud Functions programming model for composing individual functions into larger applications. Compositions, informally named apps, run in the cloud using automatically managed compute and memory resources. Composer is an extension of the function-as-a-service computing model, and enables stateful computation, control flow, and rich patterns of data flow.

Composer requires storage to manage state. We can provision storage for you, via the app init command. If you would rather start by exploring the programming model and tooling, take a look at the preview command. Below, you can explore the full command structure.

Learn about Commands

  • app preview

    Prototype a composition, with visualization help

  • app create

    Deploy a composition

  • app invoke

    Invoke a composition

  • session list

    List recent app invocations

  • session get

    Fetch the result of a session

  • session kill

    Kill a live session

  • session purge

    Purge the state for a completed session
